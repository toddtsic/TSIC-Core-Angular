Store Admin

This is where you build and run your event's merchandise store — the items families can buy (spirit wear, add-ons, extras), the colors and sizes they come in, and the sales numbers behind them. It's organized into four tabs: Items, Colors, Sizes, and Analytics.

How items and SKUs fit together

An item is a product (say, "Event Hoodie") with a name, price, and description. When you create it, you pick which colors and sizes it comes in, and the store automatically generates a SKU for every combination — Navy/Small, Navy/Medium, Red/Small, and so on. Each SKU is what actually gets sold and stocked.

Set up colors and sizes first.

An item's SKU matrix is built from the colors and sizes you select — so define those on their tabs before creating items that use them.

The tabs

  • Items — add/edit products, set price and description, choose colors/sizes, and toggle an item active. Expand any item to see its SKUs and adjust each one's stock limit or turn individual SKUs off.
  • Colors — the color options items can use. Add, rename, or remove them.
  • Sizes — the size options items can use. Add, rename, or remove them.
  • Analytics — what's selling, for reviewing store performance.

Stock and availability

  • Max-can-sell caps how many of an item (and each SKU) can be sold — your inventory ceiling.
  • Active controls whether something appears in the storefront. Deactivate an item, or a single SKU, to pull it from sale without deleting it.

Common mistakes

  • Trying to delete a color or size that's in use. If a SKU uses it, the delete is blocked — retire it by removing it from items first, or just leave it.
  • Forgetting an item is inactive. If a product isn't showing in the store, check that both it and its SKUs are active.
  • Leaving max-can-sell at a placeholder. Set it to your real inventory so you don't oversell.
