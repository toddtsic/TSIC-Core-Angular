Manage Timeslots

Not enough slots = games with nowhere to go.

Keep an eye on Games Needed vs the slots you've created — a shortfall here is what stalls a build.

What exactly is a timeslot?

One game slot: a date, a field, and a start time. When you build the schedule, each game drops into an open slot. Create enough of them, across enough fields, to hold every game your event needs.

What do Interval and Max Games control?

Interval is the gap between game starts on a field (e.g. every 60 minutes). Max Games caps how many games that field runs on the day. Together they set how many slots a field produces.

How do I know if I've made enough slots?

The screen shows Games Needed — derived from your pools and pairings — against the slots you've created. If needed is higher, add slots, extend the day, or raise Max Games until you have room for every game.

A field isn't getting any games. Why?

It probably has no timeslots for that day. A field must be both assigned (in Manage Fields) and have slots here for the scheduler to place games on it. Add slots to that field's day.

Do I have to set up every day from scratch?

No — use Bulk Clone Operations to copy a fully-configured day to other dates. Set up one day the way you want it, then clone it across the rest of the weekend.

Tip: configure fields and pairings first, then timeslots. Games Needed comes from your pools and pairings, so with those settled the needed-vs-available check here is accurate the moment you look at it.
