Manage Timeslots

Timeslots are the actual game slots — a date, a field, and a start time — that games drop into when you build. You define them here per game day, and the screen tells you whether you've made enough slots for the games your event needs.

Slots are the containers; games are what fill them.

No open slot, nowhere to place a game — so timeslot capacity is what makes (or breaks) a build.

Creating slots for a day

  • Add Date to set up a game day, then generate slots across your assigned fields.
  • The interval is the time between game starts on a field; Max Games caps how many a field runs that day.
  • Add All lays slots across every field at once; adjust individual fields afterward if needed.

Needed vs. available — the capacity check

The screen compares Games Needed (what your pairings and pools require) against the slots you've created. If needed exceeds available, the build won't have room for every game — add more slots, extend the day, or raise Max Games until you have enough. This preview is the point of the screen: catch a shortfall here, not mid-build.

Bulk clone

Most events run similar days. Use Bulk Clone Operations to copy a day's timeslot setup to other dates instead of rebuilding each one — set up Saturday once, clone it to Sunday.

Common mistakes

  • Too few slots for the games needed. Watch the needed-vs-available comparison; a shortfall means the scheduler runs out of room.
  • Slots only on some fields. If a field has no timeslots, the scheduler won't use it that day even though it's assigned.
  • Rebuilding each day by hand. Clone a configured day to the rest instead.
