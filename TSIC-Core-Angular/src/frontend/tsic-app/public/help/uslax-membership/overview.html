US Lax Membership Reconciliation

This is the roster-wide version of the membership check. It looks up every registrant's USA Lacrosse membership in one pass, refreshes the expiration date on file, flags anyone whose membership isn't in good standing, and lets you email those people a ready-made message telling them how to fix it.

The workflow

  1. Pick a role — Players or Coaches. Each loads its own eligible list (active registrants with a membership ID on file).
  2. Reconcile All — the tool pings USA Lacrosse for each candidate, updates expiry dates, and fills the grid with each person's status, age-verification, involvement, and old/new expiry.
  3. Review who needs action — rows are flagged when the check errored, the status isn't Active, there's no expiry on file, or the membership expires before this event's required date.
  4. Email them — use select rows needing action, open the compose panel (it pre-fills a role-appropriate template), and send. Each recipient gets a personalized message.
People in good standing are never emailed — even if you select them.

The server re-checks each recipient and skips anyone already Active and valid, so a broad selection won't spam families who are fine. Only those who genuinely need action receive a message.

What "needs action" means

A registrant is flagged when any of these is true:

  • The USA Lacrosse lookup errored (couldn't be checked).
  • Status is anything other than Active (Pending, Suspended, Inactive…).
  • There's no expiration date on file.
  • The membership expires before this event's valid-through date.

Other actions

  • Excel export the full reconciliation grid for your records or to share.
  • The compose panel supports personalization tokens (name, membership ID, status, expiry) that fill in per recipient — the default template already uses them, and you can edit the wording.

Common mistakes

  • Emailing before reconciling. Run Reconcile All first so statuses and expiry dates are current — otherwise you're acting on stale data.
  • Assuming selected = sent. Recipients without an email, or already in good standing, are skipped; the send summary tells you exactly who got a message.
  • Editing the template into an accusation. The default copy deliberately reports status and lets each person self-route — keep that tone so it reads correctly for every recipient.
