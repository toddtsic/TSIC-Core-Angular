Manage Fields

Pick which fields this event plays on. Fields live in a shared pool; here you pull the ones you're using into the event so scheduling can put games on them. Two side-by-side panels: Available Fields (in the pool, not yet in this event) and League-Season Fields (active for this event's scheduling).

Only assigned fields get games.

The scheduler can only place games on fields in the League-Season panel. If a field isn't there, it's not in play.

Assigning and removing

  • Select fields in Available and move them to League-Season to put them in play. Select several to assign in a batch.
  • Move a field back to remove it from the event. Use Filter fields… and the sortable columns to find one quickly.
  • A field that already has scheduled games can't be removed — clear or move its games first (in the Rescheduler), then remove it.

Field details

Open a field to edit its name, address, city/state, map coordinates, and directions. Good details here flow through to the schedules families rely on to find the field, so it's worth getting the address and directions right.

Common mistakes

  • Building a schedule with too few fields assigned. The scheduler only uses League-Season fields — assign every field you intend to play on before you build.
  • Trying to remove a field mid-event. If it has games, it won't remove; move those games first.
  • Leaving directions blank. It's the field info families see; incomplete addresses cause game-day confusion.
