Manage Pairings

Pairings are set by pool size and by rank — not per team.

One table per team-count covers every division of that size; the actual teams come from each division's ranking.

Why don't I see team names in the pairings?

Because a pairing table is defined by rank — "Rank 1 vs Rank 4" — so it can apply to every division of that pool size at once. Each division's own rankings fill in the real teams when the schedule builds.

What does "by pool size" mean?

A table is keyed to the number of teams in a pool. Your 4-team divisions all share the 4-team table, your 5-team divisions share the 5-team table, and so on. That's why you define pairings once per distinct pool size rather than once per division.

How many games will each team play?

That's the number of rounds in the table — each round is one game per team. Set it to what your event guarantees for pool play, then use Add Block if you need to extend the pattern.

The scheduler won't build a division. Could pairings be the cause?

Yes — if that division's pool size has no pairing table, there's no pattern to generate games from. Check the Scheduling Dashboard's Pairings card for pool sizes still missing a table, and fill them in.

How do I change which teams meet?

Adjust the teams' ranks (Edit Team). Since matchups are rank-based, re-ranking a team changes which rounds it lands in — that's the lever for balancing who plays whom, rather than editing individual games.

Tip: settle pool sizes (via Pool Assignment) before finalizing pairings. Change a division's team count later and it needs a different table — getting pools stable first saves reworking pairings.
