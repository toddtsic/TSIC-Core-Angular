Manage Pairings

Pairings define who plays whom — the round-by-round matchup pattern the scheduler uses to generate games. They're set up by pool size, not per team: one table for 4-team pools, one for 5-team pools, and so on. Every division of that size uses the same table.

Pairings are by rank, not by team.

A row like "Rank 1 vs Rank 4" applies to every division of that pool size — the actual teams are filled in from each division's ranking.

How a pairing table reads

  • Pick a division to view the pairing table that applies to its pool size.
  • Each round lists its matchups as Division Rank vs Division Rank — the #1 team plays the #4 team, etc.
  • Set the number of rounds to control how many games each team plays in pool play.
  • Add Block extends the pattern; Championship rows define playoff matchups sized for the pool.

Why pool size matters

Because a table is keyed to team count, defining pairings for each distinct pool size in your event is what lets the scheduler build round-robin games. The Scheduling Dashboard's Pairings card shows how many of your pool sizes already have tables — fill in any that are missing before you build.

Editing teams

You can adjust a team's rank or name from here (Edit Team). Since pairings are rank-based, a team's rank is what decides which matchups it lands in — so ranking is how you shape the strength of each round.

Common mistakes

  • Missing a pool size. If a division's size has no pairing table, the scheduler can't build its games. Cover every distinct pool size.
  • Expecting to pick teams directly. You set the rank pattern; teams flow in from each division's rankings.
  • Too few rounds. The round count is the number of pool-play games per team — set it to what your event promises.
