Bracket Seeds

This screen defines the playoff bracket — which teams feed each championship game. You don't pick team names here. Instead, each game slot is filled by a seed: a pool plus a rank, like "the #1 team out of Pool A." When pool play finishes, standings decide who that actually is, and the bracket fills itself in.

You're wiring the bracket, not choosing teams.

A slot set to "Pool A, Seed 1" resolves to whichever team finishes first in Pool A — automatically, from the standings.

How a bracket game is defined

Every bracket/playoff game has two sides. For each side you set two things:

Seed Pool

Which round-robin pool (division/age group) the team comes from.

Seed Rank

Where they finished in that pool — 1 for the winner, 2 for runner-up, and so on.

Read together, the two sides of a game say something like "Pool A #1 plays Pool B #2." Click Edit on a game to set its two seeds.

Standings are right there to guide you

The Standings panel shows each pool's current order — games played, record, points. Use it to sanity-check your seeding: the seeds you assign should line up with how the standings will resolve once pool play is complete.

Where do these games come from?

Bracket games aren't created here — they're created during scheduling as non-round-robin game types (championship, final, semi, quarter, and so on). If this page says there are no bracket games, none have been created yet. Build your bracket structure in the Schedule Hub first, then come here to seed it.

Common mistakes

  • Looking for team names. There aren't any — you assign a pool and a rank, and the team is filled in from standings later.
  • Seeding before the bracket exists. If the list is empty, create the playoff games in scheduling first.
  • Reseeding after games are played. Changing a seed changes who advances — only do it deliberately, and re-check the affected games.

Re-seed events

On a re-seed event you'll see a banner: seed pools may be drawn from any round-robin age group across the job, and each side's rank list is capped at the chosen pool's size. Everywhere else, seeding works within the event's own pools.
