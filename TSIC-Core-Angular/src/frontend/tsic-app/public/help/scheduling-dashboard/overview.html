Scheduling Dashboard

This is your scheduling home — a map of the whole process. The pipeline cards show how far along each prerequisite is, and each one links straight to the screen where you finish it. Work down the cards until they're green, then build.

Green means ready.

A card's color and count tell you what's done and what still needs setup, so you always know the next step.

Then build, then check

  • Once the pipeline is green, open Schedule Builder (the Schedule Hub) to generate the games.
  • The Post-Scheduling Tools — View Schedule, Rescheduler, Schedule QA Report — are for after the build: reviewing, moving games, and validating.

Common mistakes

  • Jumping to the builder with red cards. A build needs the whole pipeline in place — a card that isn't green is exactly what would block or thin the build.
  • Skipping a step because it "looks" done. Trust the card's count — it reflects the real data, not your memory of setting it up.
