Discount Codes

A code only works when it's active AND inside its date window.

That's the first thing to check when a registrant says their code was rejected.

A registrant says the code isn't working. What do I check?

Three things: is the code Active, is today inside its Start–End window, and did they type the Code Name exactly? An inactive or expired code — or a typo — is the usual cause.

Does "10" mean 10% off or $10 off?

It depends on the Discount Type. As a Percent code, 10 = 10% off; as an Amount code, 10 = $10 off. Set the type deliberately — on higher-priced registrations the two are very different.

Why can't I delete a code?

Because it's already been used — its Usage Count is above zero, and deleting it would remove part of the payment record. Turn its Active switch off instead; it stops working immediately but the history stays intact. Only unused codes can be deleted.

How do I turn off all discounts after a deadline?

Use deactivate all to switch every code off in one action. Going the other way, activate all turns them back on — useful for opening a promo window. You can still toggle individual codes from their rows.

How do I know how many times a code has been used?

The Usage Count column tracks redemptions per code. It's read-only — it reflects real checkouts — and it's what determines whether a code can be deleted or only deactivated.

Can I schedule a code to start and stop on its own?

Yes — that's what Start Date and End Date are for. Set an early-bird code to begin and end on specific days and, as long as it's active, it turns itself on and off with the window.

Tip: always give a promo code an End Date. A discount left active by accident quietly reduces every registration that follows — an end date closes that gap for you.
