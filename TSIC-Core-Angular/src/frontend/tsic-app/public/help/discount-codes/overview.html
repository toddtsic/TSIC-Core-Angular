Discount Codes

Create the codes registrants type in at checkout to take money off their registration. This screen lists every code for the event; add new ones, edit them, and switch them on or off. Because codes touch what people actually pay, a few details are worth getting right.

What a code is made of

  • Code Name — what the registrant types (e.g. SUMMER2026). Keep it short and easy to share.
  • Discount Type — how the reduction is calculated:
    Percent

    A share of the price — "10" means 10% off.

    Amount

    A flat sum — "10" means $10 off.

  • Start / End Date — the window the code is valid.
  • Active — the on/off switch. A code only works when it's active and inside its date window.
Percent vs. Amount is the easy one to get wrong.

"10" as a Percent is a 10% discount; "10" as an Amount is $10 off. On a $300 registration that's a $30 difference per registrant.

Usage and deleting

The Usage Count column shows how many times a code has been redeemed. Once a code has been used, you can't delete it — that would erase part of the money record. Instead, switch it off by turning Active to inactive (or letting its End Date pass). Unused codes can be deleted outright.

Activate / deactivate in bulk

The activate all / deactivate all controls flip every code at once — handy for opening a promo window or shutting all discounts off after a deadline without toggling each row.

Common mistakes

  • Leaving a code live too long. A forgotten active code keeps discounting registrations. Set an End Date so it retires itself.
  • Wrong discount type. Double-check Percent vs Amount before saving — it's the difference between 10% and $10.
  • Trying to delete a used code. You can't; deactivate it instead so the redemption history stays intact.
