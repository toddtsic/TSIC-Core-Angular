Upload Uniform Numbers

Always start from a freshly downloaded template.

The file has hidden columns the system needs to match each player. Type numbers into your own spreadsheet and the upload can't line rows up.

Which columns do I fill in?

Only UniformNo and DayGroup. Every other column — especially the ID column — must stay exactly as downloaded, since that's how each row is matched to the right player.

My upload was rejected. What's wrong?

Check three things: the file is a .xlsx (not .csv or .xls), it's under 10 MB, and its columns are unchanged from the template. Any of those will stop the whole file from being accepted.

The results show some rows as Errors. What now?

The Error Details table lists each bad row and the reason. Fix those rows in the spreadsheet and upload again — the rows that already applied stay applied, so re-uploading only catches up the ones that failed.

What does "Unchanged" mean — did those players fail?

No. "Unchanged" rows already had the number/group in the file, so there was nothing to update. It's a success, not an error — only the Errors count needs your attention.

Is it safe to upload the same file twice?

Yes. The upload just applies whatever's in the file; running it again sets the same values, so rows that already matched show as Unchanged. That's why fixing errors and re-uploading is the standard fix.

Tip: if your roster changed since you last downloaded the template, grab a fresh one — a stale file won't contain players who registered after you downloaded it.
