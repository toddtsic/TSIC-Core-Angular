Reports Library

Know the report's name? Search for it.

The search box matches names and descriptions — usually quicker than clicking through category tabs.

How are the reports organized?

Into category tabs — Rosters, Schedules, Registrations, Financials, Camp, Recruiting, Administration. Pick a tab to narrow the list, or search across all of them.

What happens when I run a report?

Most reports download a file — an Excel workbook or a PDF ready to print or share. Some open an in-app builder where you choose columns and options first, then generate the output. Everything is scoped to the event you're in.

A report I expected isn't in the list. Where is it?

The library is filtered to your event type and your role, so reports that don't apply — or that sit above your role — simply don't appear. It's not broken; if you think one should be there, a SuperUser can check what's assigned to the job.

The report came back empty or looks incomplete. Why?

Reports read live event data, so a thin or blank result almost always means the data isn't there yet — the schedule hasn't been built, teams aren't assigned, or no registrations fall in the range. Fix it at the source screen and re-run.

Is there a faster way to get to the reports I run every week?

Yes — Recents keeps your most recently run reports at the top, so the handful you use regularly are a single click away without searching or tab-hopping.

Do reports cover the whole event or just part of it?

They're scoped to the current event. A roster report covers this event's rosters, a schedule report this event's games, and so on. Switch events (jobs) to run the same report elsewhere.

Tip: run your key reports after the schedule is built and rosters are settled. Because they reflect live data, a report is only as complete as the event behind it.
