Reports Library

This is the catalogue of every report and export available for your event — rosters, schedules, registration lists, financial summaries, and more. Find the one you need, run it, and it either downloads a file or opens a builder where you choose options first.

Finding a report

  • Category tabs group the library — Rosters, Schedules, Registrations, Financials, Camp, Recruiting, Administration. Switch tabs to narrow the list.
  • Search by name or description if you know roughly what you're after — faster than browsing.
  • Recents keep your last-run reports within reach, so the ones you use every week are one click away.
You only see reports that apply to you.

The library is filtered to your event type and your role — a lacrosse recruiting report won't clutter a camp's list, and reports above your role simply don't appear.

Running a report

Click a report to run it. Most produce a download — an Excel workbook or a PDF you can print or share. A few open an in-app builder instead (for example, a roster or schedule-list designer), where you pick columns and options and then generate the output. Either way, the report is scoped to this event.

Common mistakes

  • Hunting through tabs for a report you know by name. Just search — it matches names and descriptions.
  • Expecting a report that isn't listed. If it's missing, it's usually filtered out by event type or role — not broken. A SuperUser can confirm what's assigned.
  • Running before the data exists. A schedule or roster report only reflects what's been built — generate the schedule and settle rosters first.

If a report looks empty or wrong

Most reports pull live event data, so a blank or thin result usually means the underlying data isn't there yet (unbuilt schedule, unassigned teams, no registrations in range) rather than a problem with the report itself. Check the source screen, then re-run.
