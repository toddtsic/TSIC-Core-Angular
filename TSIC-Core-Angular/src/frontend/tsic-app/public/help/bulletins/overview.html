Bulletins

Bulletins are the announcements that appear on your event's home page — deadlines, weather calls, check-in reminders, anything your families need to see. This screen lists every bulletin; click one to edit it, or add a new one.

These are public.

An active bulletin inside its date window shows to everyone who visits the event home page — so write it for that audience.

What makes up a bulletin

  • Title and Content — the headline and the message body (rich text).
  • Start Date / End Date — the window it's shown. Outside that window it stays hidden even if it's active.
  • Active switch — the on/off. A bulletin only appears when it's both active and inside its dates.

Two gates decide visibility

Active + Within date window = Visible on home page

If a bulletin isn't showing, check both: is the Active switch on, and is today between its Start and End dates?

Tokens and preview

You can drop tokens into the content to pull in dynamic bits of text. The Resolved Preview shows the bulletin as visitors will actually see it — and the Simulate checkboxes let you preview how it looks under different conditions, since some content only appears for certain events or registrants. Always glance at the preview before saving.

Drafting help

Not sure how to word it? Describe the bulletin you want ("remind everyone check-in opens at 7am Saturday") and the editor can draft the title and content for you to refine.

Common mistakes

  • Saved but not showing. Almost always the Active switch is off, or today falls outside the date window.
  • End before start. The date window has to run forward — the editor flags it if you reverse them.
  • Editing a live bulletin without previewing. It's public the moment it's active and in-window; use the Resolved Preview so families don't see a half-finished edit.
